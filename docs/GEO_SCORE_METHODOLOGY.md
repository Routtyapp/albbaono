# GEO Score 측정 방법론

## 개요

GEO(Generative Engine Optimization) Score는 웹사이트가 AI 검색 엔진(ChatGPT, Gemini, Perplexity 등)에서 얼마나 잘 인용될 수 있는지를 측정하는 점수입니다.

**총점: 100점 만점**

| 카테고리 | 배점 | 비중 |
|---------|------|------|
| 구조 (Structure) | 25점 | 25% |
| 스키마 (Schema) | 25점 | 25% |
| URL | 15점 | 15% |
| 메타태그 (Meta) | 20점 | 20% |
| 콘텐츠 (Content) | 15점 | 15% |

---

## 등급 체계

| 등급 | 점수 범위 | 설명 |
|-----|----------|------|
| A+ | 95-100 | AI 검색 엔진 최적화 완벽 |
| A | 85-94 | AI 검색 엔진 최적화 우수 |
| B+ | 78-84 | AI 검색 엔진 최적화 양호 |
| B | 70-77 | AI 검색 엔진 최적화 보통 |
| C+ | 63-69 | 개선 필요 |
| C | 55-62 | 상당한 개선 필요 |
| D | 40-54 | 많은 개선 필요 |
| F | 0-39 | 전면 개선 필요 |

## 1. 구조 분석 (Structure) - 25점

AI는 명확하게 구조화된 콘텐츠를 선호합니다. 목록형 콘텐츠가 AI 인용의 50%를 차지합니다.

### 1.1 목록형 콘텐츠 (7점)

| 조건 | 점수 |
|-----|------|
| `<ul>`, `<ol>` 3개 이상 + `<li>` 10개 이상 | 7점 |
| `<ul>`, `<ol>` 2개 이상 + `<li>` 5개 이상 | 5점 |
| `<ul>`, `<ol>` 1개 이상 + `<li>` 3개 이상 | 3점 |
| 목록 없음 | 0점 |

**측정 방법:**
```javascript
const ulCount = $('ul').length;
const olCount = $('ol').length;
const liCount = $('li').length;
```

### 1.2 표 형식 사용 (6점)

표 형식 콘텐츠는 AI 인용률을 2.5배 높입니다.

| 조건 | 점수 |
|-----|------|
| `<thead>` 또는 `<th>` 포함 테이블 2개 이상 | 6점 |
| `<thead>` 또는 `<th>` 포함 테이블 1개 | 4점 |
| 일반 테이블 존재 | 2점 |
| 테이블 없음 | 0점 |

**측정 방법:**
```javascript
const tables = $('table');
tables.each((_, table) => {
  const hasThead = $(table).find('thead').length > 0;
  const hasTh = $(table).find('th').length > 0;
});
```

### 1.3 두괄식 작성 (7점)

각 섹션의 첫 40-60단어 안에 핵심 답변이 있으면 AI 추출 확률이 높아집니다.

| 조건 | 점수 |
|-----|------|
| 70% 이상 섹션이 두괄식 | 7점 |
| 50% 이상 섹션이 두괄식 | 5점 |
| 30% 이상 섹션이 두괄식 | 3점 |
| 30% 미만 | 1점 |

**측정 방법:**
- 각 `<h1>`~`<h6>` 다음 요소의 첫 60단어 분석
- 숫자, 정의형 문장, 결론 키워드 존재 여부 확인

### 1.4 콘텐츠 최신성 (5점)

AI 봇 트래픽의 65%는 최근 1년 내 발행된 콘텐츠로 향합니다.

| 조건 | 점수 |
|-----|------|
| 1년 이내 발행/수정 날짜 확인됨 | 5점 |
| 날짜 표기 있으나 1년 초과 | 2점 |
| 날짜 표기 없음 | 0점 |

**측정 방법:**
```javascript
// 확인하는 위치
$('meta[property="article:published_time"]')
$('meta[property="article:modified_time"]')
$('time[datetime]')
// JSON-LD의 datePublished, dateModified
```

## 2. 스키마 분석 (Schema) - 25점

AI는 구조화된 데이터(Schema.org)를 통해 정보를 정확히 파악합니다.

### 2.1 Product 스키마 (10점)

| 조건 | 점수 |
|-----|------|
| 모든 권장 필드 포함 (6개 이상) | 10점 |
| 4개 이상 필드 포함 | 8점 |
| 2개 이상 필드 포함 | 5점 |
| 기본 필드만 존재 | 3점 |
| 없음 | 0점 |

**필수 필드:** `name`, `description`
**권장 필드:** `image`, `brand`, `sku`, `offers` (price, priceCurrency, availability)

### 2.2 FAQ 스키마 (5점)

| 조건 | 점수 |
|-----|------|
| FAQPage + Q&A 5개 이상 | 5점 |
| FAQPage + Q&A 3개 이상 | 4점 |
| FAQPage 존재 | 2점 |
| 없음 | 0점 |

### 2.3 HowTo 스키마 (5점)

| 조건 | 점수 |
|-----|------|
| HowTo + step 5개 이상 | 5점 |
| HowTo + step 3개 이상 | 4점 |
| HowTo 존재 | 2점 |
| 없음 | 0점 |

### 2.4 Review 스키마 (5점)

| 조건 | 점수 |
|-----|------|
| AggregateRating + ratingValue + reviewCount | 5점 |
| AggregateRating만 존재 | 3점 |
| Review만 존재 | 2점 |
| 없음 | 0점 |

## 3. URL 분석 - 15점

검색엔진은 URL을 페이지의 첫 번째 레퍼런스로 사용합니다.

### 3.1 하이픈 사용 (5점)

| 조건 | 점수 |
|-----|------|
| 언더스코어(_) 없음, 하이픈(-) 사용 | 5점 |
| 언더스코어 포함 | 0점 |

**이유:** 구글은 하이픈을 띄어쓰기로 인식하지만, 언더스코어는 연결 문자로 인식합니다.
- `/seo-guide` → "seo guide" ✓
- `/seo_guide` → "seoguide" ✗

### 3.2 소문자 사용 (5점)

| 조건 | 점수 |
|-----|------|
| URL 전체 소문자 | 5점 |
| 대문자 포함 | 0점 |

**이유:** 일부 서버는 대소문자를 구분하여 `/SEO`와 `/seo`를 다른 페이지로 인식할 수 있습니다.

### 3.3 영문 URL (5점)

| 조건 | 점수 |
|-----|------|
| 인코딩 문자(%XX) 없음 | 5점 |
| 한글/특수문자 인코딩 포함 | 0점 |

**이유:** 한글 URL `/검색`은 `/%EA%B2%80%EC%83%89`로 인코딩되어 공유 시 깨지고 검색엔진이 읽기 어렵습니다.

## 4. 메타태그 분석 (Meta) - 20점

### 4.1 Title 최적화 (7점)

| 조건 | 점수 |
|-----|------|
| 50-60자 + 쇼핑 의도 키워드 | 7점 |
| 50-60자 | 5점 |
| 40-70자 | 4점 |
| 20-40자 또는 70자 초과 | 2점 |
| 20자 미만 | 1점 |

**쇼핑 의도 키워드:** 추천, 비교, 리뷰, 가격, 구매, 최고, 베스트, 가이드, 방법, 사용법

### 4.2 Description 최적화 (7점)

| 조건 | 점수 |
|-----|------|
| 120-160자 + 의도 문구 | 7점 |
| 120-160자 | 5점 |
| 100-180자 | 4점 |
| 50-100자 또는 180자 초과 | 2점 |
| 50자 미만 | 1점 |

**의도 문구:** "~을 찾는", "~를 위한", "~보다", "추천", "비교", "방법"

### 4.3 Open Graph (3점)

| 조건 | 점수 |
|-----|------|
| og:title, og:description, og:image 모두 존재 | 3점 |
| 2개 존재 | 2점 |
| 1개 존재 | 1점 |
| 없음 | 0점 |

### 4.4 Canonical URL (3점)

| 조건 | 점수 |
|-----|------|
| `<link rel="canonical">` 존재 + 현재 URL과 일치 | 3점 |
| canonical 존재 (다른 URL 지정) | 2점 |
| 없음 | 0점 |

## 5. 콘텐츠 분석 (Content) - 15점

### 5.1 데이터/통계 포함 (5점)

AI는 추상적 표현보다 구체적 통계와 데이터를 신뢰합니다.

| 조건 | 점수 |
|-----|------|
| 통계/수치 데이터 10개 이상 | 5점 |
| 통계/수치 데이터 5개 이상 | 4점 |
| 통계/수치 데이터 2개 이상 | 2점 |
| 없음 | 0점 |

**측정 패턴:**
- 퍼센트: `50%`, `72.5%`
- 숫자+단위: `100만`, `2.5배`, `30억`
- 연도: `2024년`, `2023`
- 통계 키워드: 통계, 조사, 연구, 리포트, 보고서

### 5.2 출처 표기 (5점)

| 조건 | 점수 |
|-----|------|
| 신뢰 도메인 참조 2개 이상 또는 인용 점수 10 이상 | 5점 |
| 인용 점수 5 이상 | 4점 |
| 외부 링크 2개 이상 또는 blockquote 존재 | 2점 |
| 없음 | 0점 |

**신뢰 도메인:** wikipedia.org, gov.kr, go.kr, ac.kr, edu, bloomberg.com, reuters.com, statista.com

**인용 점수 계산:**
```
외부 링크 수 (최대 5) + blockquote × 2 + cite × 2 + 신뢰 도메인 참조 × 2
```

### 5.3 HTML vs 이미지 (5점)

AI는 이미지 내 텍스트를 읽을 수 없습니다. HTML 구조화가 필수입니다.

| 조건 | 점수 |
|-----|------|
| 텍스트 1000자 이상 + 표/목록 2개 이상 | 5점 |
| 텍스트 500자 이상 + 표/목록 1개 이상 | 4점 |
| 텍스트 300자 이상 | 3점 |
| 텍스트 부족 | 2점 |
| 인포그래픽 의존도 높음 | 1점 |

## 서브페이지 분석

서브페이지 포함 분석 시, 각 페이지의 점수를 개별 계산 후 평균을 산출합니다.

```
최종 점수 = (페이지1 점수 + 페이지2 점수 + ... + 페이지N 점수) / N
```

## 개선 권장사항 우선순위

권장사항은 해당 항목의 최대 점수에 따라 우선순위가 결정됩니다.

| 우선순위 | 최대 점수 | 예시 |
|---------|----------|------|
| 높음 (High) | 7점 이상 | Product 스키마, Title, Description, 목록형 콘텐츠, 두괄식 |
| 중간 (Medium) | 5-6점 | 표 형식, FAQ/HowTo/Review 스키마, 최신성, 통계, 출처 |
| 낮음 (Low) | 5점 미만 | Open Graph, Canonical URL |

## 참고 자료

- AI 봇 트래픽의 65%는 최근 1년 내 발행된 콘텐츠로 향함
- 목록형 콘텐츠가 AI 인용의 50% 차지
- 표 형식은 인용률을 2.5배 향상
- 오리지널 통계/연구 결과는 30-40% 더 높은 가시성
- 소비자 58%가 생성형 AI로 제품 추천 받음
- AI 추천 기반 직접 유입 전년 대비 8배 증가
